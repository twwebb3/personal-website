Introduction

There are many different offerings of git: GitHub, GitLab, Bitbucket, and more. I predominantly use GitHub and Bitbucket, but I have used CodeCommit in the past. The fundamentals are the same across all platforms, but the user interface and additional features differ. Below are notes on the fundamentals. First though, what is git? git is a distributed version control system that allows for collaboration, and is massively helpful with tracking changes in code (and reverting to the last working version when you break something).

.gitignore

This is a file telling git what not to upload to the repository from your local machine, it should include everything that you do not want publicly available and then potentially some more. In a .gitignore file you simply list files and directories you do not want tracked by git. Every file or directory on its own line. Below is a list of things I typically include in a .gitignore:

  • venv/
  • .idea/
  • __pycache__/
  • .DS_Store
  • *.sqlite3
  • .env
  • data/
  • *.xlsx
  • *.csv

For reference, '*.file_extension' ignores every file with that extension, and 'directory/' ignores every thing in that directory. This list isn't exhaustive, and different types of projects will have varying needs for what to include in a .gitignore.
